# AIコーディング支援ツールの利用戦略とサブスクリプション最適化に関する分析 | 2025-11-08

## 1. 序論

本文書は、利用中の複数のAIコーディング支援ツール（Gemini、Roo Code、GitHub Copilot）の利用実態、コスト、および機能制限の分析に基づき、実施したサブスクリプション戦略の変更（GitHub Pro+ への移行）について、その分析と今後の展望をまとめたものです。

課題は、メインツールである Gemini CLI の実質的な利用制限と、バックアップツールである Roo Code の高額な従量課金コストでした。この課題を解決するため、GitHub Copilot の Pro+ プランへアップグレードを実施しました。

## 2. 現状の利用ツールと詳細な課題分析

主に3つのAIコーディングツールを併用していますが、それぞれに利点と制約があります。

### 2.1. Gemini (Code Assist / CLI)

* **利用形態:** 当初 `Gemini Code Assist` を利用していましたが、セッションが長くなると重たくなる問題がありました。現在は、その問題がない `Gemini CLI` をメインツールとして利用しています。
* **課題: 実質的なリクエスト上限:**
    * 有料アカウントの公式な利用上限は「**1日1,500回**」とされています。
    * しかし、Gemini CLI はユーザーのリクエスト数だけでなく、その**内部処理もカウントする**仕様のようです。
    * その結果、`Gemini CLI` の `/stats` コマンドで確認すると、実際には「**100回を少し超える**」程度で、1日の利用制限に達してしまいます。
* **業務への影響:**
    * 「**Nekote**」の開発作業において、全稼働日の約**8割**において、**5～7時間**のコーディングでこの上限に達します。
    * （子どものお迎えの都合上、1日の総労働時間は**7時間半**）就業時刻まで Gemini を継続して利用できる日は、全体の約**2割**に留まり、多くの場合、作業が途中で中断する原因となっていました。

### 2.2. Roo Code + Anthropic (Claude)

* **利用形態:** Gemini CLI が利用上限に達した際の、バックアップツール。Anthropic の API を Pay-as-you-go（従量課金）で利用。
* **課題: 利用コスト:**
    * バックアップとしての利用がメインであるにもかかわらず、「**先月の後半**」の半月だけで「**60ドル**」の費用が発生しました。
    * これは、月額換算で「**120ドル**」規模になる可能性を示しており、コスト負担が大きい状態でした。
* **ツールの特性:**
    * Roo Code は、依頼していないテストコードの出力など、時に意図しない（余計な）動作をすることがあります。

### 2.3. GitHub Copilot (Proプラン)

* **当初の評価:**
    * 「**GPT-4.1**」が（基本的に）無制限で利用可能である点は評価していました。
    * しかし、Proプランでは「プレミアムリクエスト」が「**月300回**」と、Gemini の実質的な上限（1日100回強）と比較しても著しく少なく感じられました。
* **課題: 利用習慣の欠如:**
    * 「**2～3日**」で使い切ってしまうことを恐れ、本当に必要な時だけ使おうとリクエストを「温存」していました。
    * その結果、Copilot を積極的に使う習慣が形成されず、ツールを有効活用できていませんでした。

## 3. 新戦略：GitHub Pro+ へのアップグレード（実施済み）

上記の課題を解決するため、GitHub Copilot のプランを Pro から Pro+ へ変更しました。

### 3.1. 発見と変更点

* **経緯:** 「**GitHub Spark**」を試す過程で、Pro+ プランの存在を知りました。
* **プラン内容:** Pro から Pro+ へのアップグレードにより、プレミアムリクエストの上限が「月300回」から「**月1,500回**」へと大幅に増加しました。
* **コスト:**
    * Pro+ の月額費用は「**39ドル**」です。
    * これは、Pro プランからの差額（増額）が「**29ドル**」であることを意味します。
    * ブラウザ版の ChatGPT や Gemini の上位版が「月200ドル」程度することと比較しても、安価です。

### 3.2. リクエスト回数の比較と実用性の評価

* **カウント方式の違い:** Gemini が内部処理をカウントするのに対し、GitHub のプレミアムリクエストは**内部処理をカウントしない**ようです。これは、実際に大規模なコードレビューをいくつかのやり取りを交えて行った後も、オンラインの統計上、**数回しかカウントが消費されていなかった**という実体験に基づいています。
* **実質的な利用回数:** 「**月1,500回**」は、1日あたり「**約70回**」のプレミアムリクエストが使用可能であることを意味します。
* **評価:** これは、Gemini の実質的な上限（「**100回強**」）には劣るものの、Gemini と併用する**主力ツールの一つ**として、十分に実用的なリクエスト回数であると判断できます。

### 3.3. 期待される効果と今後の観測

1.  **コスト削減:**
    * 月額「**29ドル**」の追加投資（Pro+へのアップグレード）により、月額「120ドル」規模に達していた **Roo Code + Claude の利用料を「1/3」程度に削減できると期待しています**。
    * これにより、コードの品質を落とすことなく、AIツール利用の総コストを大幅に削減できる見込みです。

2.  **新しいワークフローの確立:**
    * 「Gemini CLI が上限に達したら、GitHub Copilot を使う」
    * あるいは、「初めから両者をバランス良く併用する」
    * 上記のような、柔軟で持続可能な新しいワークフローが確立されました。

3.  **Roo Code の戦略的利用:**
    * 今後も、Gemini や Copilot では解けない複雑な問題については、Roo Code を利用する可能性はゼロにはならないでしょう。
    * ただし、その利用はコストを意識し、戦略的に判断します。

## 4. 各ツールの戦略的再定義

今回の戦略変更に伴い、各ツールの役割を以下のように再定義します。

ただし、これは Roo Code + Claude と Gemini の優劣を論じるものではありません。タスクによって得手不得手があり、**両者はほぼ同等の能力を持つ**と考えています。

あくまで個人的な印象ですが、Roo Code + Claude は、直感的なアプローチでコードの要素を組み合わせて機能させる「**アーティスト**」的な側面を持ち、Gemini は、全体を体系的に計画し、ロジカルに分解して実装する「**弁護士**」的な側面を持っているように感じられます。

* **Gemini (CLI):**
    * 複雑なタスクほど高い能力を発揮しますが、時に回りくどい回答をすることがあります。実質的な利用回数の制限が厳しく、無計画に使うと1日の途中で利用不能になり、作業が中断するリスクを伴うツール。
    * **（主力：高難易度・計画的タスク担当）**

* **GitHub Copilot (Pro+):**
    * 今後は、基本的に高度なモデル（プレミアムリクエスト）を常時利用していきます。その上で、プレミアムリクエストの消費を抑えるため、明らかに軽微な作業だと判断できる場合にのみ、意識的にGPT-4.1（標準モデル）に切り替えて運用します。
    * **（主力：高度なタスク担当 ＆ 補助：軽作業によるリクエスト節約）**

* **Roo Code + Claude:**
    * 高コストと引き換えに、高品質なコードを生成する、柔軟な対応が可能なツール。
    * **（補完的バックアップ：特定の問題解決・別のアプローチ試行用）**

## 5. 結論

GitHub Pro+ へのアップグレードは、Gemini の「実質的な日次制限」と Roo Code の「高額な従量課金」という2つの主要な問題を同時に解決するための、俯瞰的に見て合理的な判断でした。

これにより、開発の質を維持・向上させながら、AIツールの利用コストを大幅に削減することにつながると期待されます。今後は、この新しい体制でのコストと生産性の変化を観測していきます。
